## category 表乱码修复

- **问题**：本地数据库 zhuxian_wiki 的 category 表 id 1-7 的 name 字段出现乱码
- **根因**：数据写入时 MySQL 连接使用了 Windows-1252 字符集，UTF-8 中文字节被当成 cp1252 字符处理后二次 UTF-8 编码存入
- **修复方法**：通过 HEX 提取原始字节，cp1252 反向解码还原，直接 UPDATE 写回正确 UTF-8 值
- **修复内容**：
  - id=2: 职业
  - id=3: 攻略
  - id=4: 副本
  - id=5: 装备
  - id=6: 宠物
  - id=7: 坐骑
- **备份表**：category_backup_20260501（已创建）
- **id=1 name="1"**：疑似测试数据，用户待确认
